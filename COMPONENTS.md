## iOS Design Considerations

All components follow these principles:

1. **Touch Targets**: Minimum 44x44pt for interactive elements
2. **Dynamic Type**: Full support for accessibility text sizes
3. **Dark Mode**: Semantic colors that adapt automatically
4. **Safe Areas**: Respect notch, home indicator, keyboard
5. **Haptic Feedback**: Sensory feedback for key interactions
6. **Gestures**: Native swipe, long-press, pull-to-refresh
7. **Accessibility**: VoiceOver, reduce motion, increase contrast

## Contributing

When adding new components:

1. Follow existing patterns in `Tokens.swift` and `AppIcon.swift`
2. Use semantic token values (not hardcoded colors/sizes)
3. Support all standard variants and sizes
4. Include accessibility labels
5. Add to `registry.json` with dependencies
6. Test on iPhone and iPad
